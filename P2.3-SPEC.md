# P2.3-SPEC.md — 接真 AI（DeepSeek）+ A/B 两种思考

> 给 Codex 的实现规格。配合 AGENTS.md、DAILY-LOOP-SPEC.md 一起读，硬约束继续生效。
> 触发条件：用户已决定用 **DeepSeek** 破局（Anthropic key 暂未解锁）。
> 目标：把 brain two 从「我伺候的数据库」变成「帮我想的大脑」。这一步是整个项目的价值核心。

## 0. 为什么这步绕不过后端
DeepSeek key 是一把能花钱的钥匙。前端是公开静态站（GitHub Pages）+ 公开仓库，**key 写进前端几分钟就被爬走盗刷**。所以必须加一个独立小代理藏 key：前端只调自己的代理，代理拿 key 调 DeepSeek。**「纯静态单文件」阶段到此结束**——这是接真 AI 的必然代价，不是过度工程。

## 1. 后端代理（新增，独立件）
- 形态：**Cloudflare Worker**（推荐，一个文件、免费额度足）。n8n / Supabase Edge Function 亦可。
- 职责：接收前端 POST → 注入 `DEEPSEEK_API_KEY`（存 Worker 加密变量，**不进任何前端 / 仓库**）→ 调 DeepSeek `chat/completions` → 返回结构化结果。
- CORS：只允许来自 `k1mzee.icu`。
- 两个路由（或一个路由 + mode 字段）：`/review`（模式 A）、`/decide`（模式 B）。
- 前端用一个**可配置 endpoint** 指向它；mock 与真之间只切这个地址，UI 零改动。

## 2. 两种思考（产品核心）

### A · 每日复盘（替我想）
- 输入：当天 `day`（entries + fitness）。
- 输出：`review { summary, score{overall, dims}, advice[], promotedCardIds[] }`（结构沿用 DAILY-LOOP-SPEC 1.4）。
- 行为：读当天原始记录 → 提炼「今天真正重要的是什么 / 在重复什么模式 / 下一步该干嘛」→ 打分 + 1-3 条建议 → 把耐用洞察提炼成卡写进卡库（personal / business 按内容归库）。

### B · 决策助手（用我的历史帮我决策）
- 输入：用户的一个具体问题（如「这个 SKU 要不要上」）。
- 检索：从**业务库现有卡**里取相关卡（先用关键词 / 简单匹配选 N 张喂进 prompt；向量检索是后续优化，本期不强求）。
- 输出：`decision { answer, basedOnCardIds[], caveats[] }`——基于**用户自己过去的判断**给建议，并标出引用了哪几张卡。
- 关键：B 的价值在于「**用你想过的东西帮你看这件事**」，所以 prompt 必须强制它**只基于喂进去的卡 + 问题**推理，引用具体卡，不够就说不够，不许凭空发挥。

## 3. Prompt 原则（写进系统提示）
- **建设性打分**：看趋势、对用户自己的基线，不做每日审判；既不吹捧也不羞辱。（沿用 DAILY-LOOP-SPEC §4）
- **扎根本人语料**：B 必须基于用户自己的卡推理并引用卡 id；A 提炼的洞察要忠于当天记录，不注水。
- **诚实**：信息不足直接说不足，不编。
- 中文输出。返回严格 JSON（前端好解析），无 JSON 外的多余文字。

## 4. 本期范围（Codex 做 / 暂缓）
**做：**
- Cloudflare Worker 代理（含 `/review`、`/decide`），key 走加密变量。
- 前端：把 A 的 `requestReview` 从 mock 切到真 endpoint（endpoint 可配置）；**新增 B「决策助手」入口**（一个输入框 + 触发，调 `/decide`，渲染 answer + 引用卡）。
- A 的「提炼洞察 → 写卡」落地（写进对应库，标 source = 当天）。
**暂缓：**
- 向量检索（B 现在用关键词选卡即可）。
- n8n 夜间自动复盘定时（P2.4）。

## 5. 硬边界
- key 永不进前端 / 仓库；Worker 变量里存。提交前扫一遍仓库无 key。
- 隐私红线：个人库 / 日志 / 复盘属个人·本地，**不随站部署、不进公开仓库**；B 检索个人库时结果只在本地呈现。代理若需落数据，个人数据不落公开存储。
- 不破坏现有功能（卡片库、3D、命令面板、今日区块、导入导出）。
- 前端保持单文件可静态部署；代理独立。
- 超出本 spec 的结构改动先停下来问人。

## 6. 验收标准（Claude 审核对照）
- [ ] Worker 部署后，前端切到真 endpoint，A 能对真实当天数据出**真复盘**（非 mock），summary/score/advice 合理且建设性。
- [ ] B：给一个决策问题，能检索到相关业务卡、给出基于这些卡的建议、并列出引用的卡 id；卡不足时如实说明。
- [ ] 仓库与前端**任何位置都无 key**；CORS 限定本站。
- [ ] 个人数据不进公开仓库 / 不落公开存储。
- [ ] A 提炼的洞察正确写入对应库、source 指向当天。
- [ ] 原有功能与移动端未被破坏；JS 无报错。
